New Workflow Rule

Ops: eq, neq, in, not_in, contains, gt, lt, gte, lte, regex, is_set, is_not_set
Types: set_priority, set_category, set_status, set_team, assign_agent, add_tag, add_note, send_notification, call_webhook, set_sla, set_custom_field, escalate

How It Works

1. Triggers — when an event fires (e.g. ticket_created), the engine finds all active rules for that event.

2. Conditions — every condition must match (AND logic). Empty conditions = always match.

3. Actions — executed in order. Failed actions are logged but don't stop the rule.

4. Audit — every execution is logged in workflow_executions for traceability.

Safety — max 50 actions per rule, infinite-loop prevention, 10ms condition timeout.

Example: Auto-escalate Critical tickets

{
  "triggerEvent": "ticket_created",
  "conditions": [
    {"field": "priority", "op": "eq", "value": "Critical"}
  ],
  "actions": [
    {"type": "add_tag", "params": {"tagName": "critical"}},
    {"type": "send_notification", "params": {"event": "critical_priority", "message": "Critical ticket created"}},
    {"type": "escalate", "params": {"level": 1}}
  ]
}
